WHAT IS HOISTING?
hoisting is a javascript default behaviour in which all the declarations 
move to the top during memory creation phase

var a;    (declaration is moved to top)

Console.log(a+10) ---> NaN (not a number)
console.log(a)    ---> Undefined
var a=10;   ---> initialisation 



JAVASCRIPT ka code alag alag phases me top to bottom do baar chalta hai
global execution context ---> aisi jagah jaha sabko(variables, functions) memory milti hai aur code execute hota hai
execution context do phase me divided hai

    MEMORY CREATION PHASE                              CODE EXECUTION PHASE
        |-----------|                                   log(a+10) = undefined +10 -> NaN
        | undefined |                                   log(a) = undefined
        |-----------|                                   a=10
              a                                        log(a)=10


is let or const variable also hoisted??
    yes they are also hoisted but they are not initialised during memory execution phase,
    their value is initialised during code initialisation phase
